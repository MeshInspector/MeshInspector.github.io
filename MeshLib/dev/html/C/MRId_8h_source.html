187// Allow constructing from `int` and other integral types.
188// This constructor is written like this instead of a plain `Id(int)`, because we also wish to disable construction
189// from other unrelated `Id<U>` specializations, which themselves have implicit conversions to `int`.
190// We could also achieve that using `template <typename U> Id(Id<U>) = delete;`, but it turns out that that causes issues
191// for the `EdgeId::operator UndirectedEdgeId` below. There, while `UndirectedEdgeId x = EdgeId{};` compiles with this approach,
192// but `UndirectedEdgeId x(EdgeId{});` doesn't. So to allow both forms, this constructor must be written this way, as a template.
193// The `= int` is there only to make the bindings emit this constructor, I don't think it affects anything else.
195MRC_API MR_UndirectedEdgeId MR_UndirectedEdgeId_Construct_int(int i);
