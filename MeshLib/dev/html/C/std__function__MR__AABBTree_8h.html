Assign a function with an extra user data pointer. Parameter userdata_callback can be null. Pass null if you don't need custom behavior when destroying and/or copying the functor. Parameter _this can not be null. It is a single object. Callback parameter _return_pass_by is an output parameter. It's will never be null, and initially points to a zeroed variable. How to use userdata_callback: The _this_userdata parameter will never be null. If *_this_userdata is non-null and _other_userdata is null, the functor is being destroyed. Perform any cleanup if needed. If *_this_userdata is null and _other_userdata is non-null, a copy of the functor is being constructed. Perform copying if needed and write the new userdata to *_this_userdata. If *_this_userdata is non-null and _other_userdata is non-null, the functor is being assigned. The simplest option is to destroy *_this_userdata first, and then behave as if it was null.
