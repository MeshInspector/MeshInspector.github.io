typedef struct MR_std_shared_ptr_MR_SphereObject MR_std_shared_ptr_MR_SphereObject

Wraps a pointer to a single shared reference-counted heap-allocated MR::SphereObject. Supported MR_PassBy modes: MR_PassBy_DefaultConstruct, MR_PassBy_Copy, MR_PassBy_Move (and MR_PassBy_DefaultArgument and MR_PassBy_NoObject if supported by the callee).

MRC_API void MR_std_shared_ptr_MR_SphereObject_Assign ( MR_std_shared_ptr_MR_SphereObject * _this,
MR_SphereObject * ptr )

Overwrite the existing instance, taking ownership of an existing pointer. The previously owned object, if any, has its reference count decremented. Parameter _this can not be null. It is a single object. Parameter ptr should point to a single object rather than to an array. Parameter ptr takes ownership of the passed pointer (if not null), and will later call MR_SphereObject_Destroy() on it automatically.

◆ MR_std_shared_ptr_MR_SphereObject_UseCount()

MRC_API int MR_std_shared_ptr_MR_SphereObject_UseCount ( const MR_std_shared_ptr_MR_SphereObject * _this)

How many shared pointers share the managed object. Zero if no object is being managed. This being zero usually conincides with MR_std_shared_ptr_MR_SphereObject_Get() returning null, but is ultimately orthogonal. Note that in multithreaded environments, the only safe way to use this number is comparing it with zero. Positive values might change by the time you get to use them. Parameter _this can not be null. It is a single object.
