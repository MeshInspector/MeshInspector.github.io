188// Allow constructing from `int` and other integral types.
189// This constructor is written like this instead of a plain `Id(int)`, because we also wish to disable construction
190// from other unrelated `Id<U>` specializations, which themselves have implicit conversions to `int`.
191// We could also achieve that using `template <typename U> Id(Id<U>) = delete;`, but it turns out that that causes issues
192// for the `EdgeId::operator UndirectedEdgeId` below. There, while `UndirectedEdgeId x = EdgeId{};` compiles with this approach,
193// but `UndirectedEdgeId x(EdgeId{});` doesn't. So to allow both forms, this constructor must be written this way, as a template.
194// The `= int` is there only to make the bindings emit this constructor, I don't think it affects anything else.
